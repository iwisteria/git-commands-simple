🚀 Git/GitHub 実用コマンド集

開発者のための完全ガイド

リポジトリの初期化・クローン

git init
新しいリポジトリを初期化
git clone
リモートリポジトリをクローン
git clone -b
特定のブランチをクローン

ファイルの追加・コミット

git add .
全ての変更をステージング
git commit -m "message"
変更をコミット
git commit -am "message"
add + commit を同時実行

ブランチの作成・切り替え

git branch
ブランチ一覧を表示
git checkout -b
ブランチ作成と切り替え
git switch
ブランチ切り替え
git branch -d
ブランチ削除

マージ

git merge
ブランチをマージ
git merge --no-ff
マージコミットを作成

リモート設定

git remote -v
リモート一覧を表示
git remote add origin
リモートリポジトリを追加

プッシュ・プル

git push -u origin
初回プッシュ
git pull
リモートから取得
git fetch --all
全ブランチをフェッチ

状態確認

git status
作業状態を表示
git diff
変更差分を表示
git diff --staged
ステージング差分を表示

履歴確認

git log --oneline
コミット履歴を1行表示
git log --graph --oneline --all
グラフ形式で履歴表示

変更の取り消し

git restore
ファイルの変更を取り消し
git restore --staged
ステージングを取り消し
git reset --hard HEAD~1
直前のコミットを取り消し

Stash（一時保存）

git stash
変更を一時保存
git stash pop
stashを適用して削除
git stash list
stash一覧を表示

タグ

git tag -a -m "message"
注釈付きタグを作成
git push origin --tags
全タグをプッシュ

リベース

git rebase
ブランチをリベース
git rebase -i HEAD~3
インタラクティブリベース

検索・調査

git grep "text"
ファイル内容で検索
git blame
変更者を特定

💡 効率化のTips

  • 頻繁に使うコマンドはエイリアスに設定する
  • .gitignore を活用して不要なファイルをコミットしない
  • コミットメッセージは明確で簡潔に書く
  • 小さな単位で頻繁にコミットする
  • プッシュする前に git status と git log で確認する
